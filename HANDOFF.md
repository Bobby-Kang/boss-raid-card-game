## ⚠️ 가장 중요한 미해결 사안 — 아트 제약

**이게 최종 도달점을 결정합니다. 다음 세션은 이 결정부터 하세요.**

레퍼런스 룩은 **배경 1장 + 투명 배경 캐릭터 컷아웃**이 전제입니다.
우리는 **각자 자기 배경(던전 벽)을 품은 사각 일러스트 2장**(`warrior_face.png` / `bugbear_face.png`)이라:

- 나란히 놓으면 **가운데 이음새**가 그대로 보임
- 그림이 꽉 차 있어 **HUD를 위에 겹칠 수 없음** (겹치면 "그림에 UI를 덧댄" 꼴 — 실제로 투기 바가 버그베어 가슴팍에 얹혀서 무대/HUD 경계를 맞추는 걸로 타협함)

**즉 지금 구조로는 "두 그림 + 그 아래 HUD"가 한계입니다.** 더 밀어붙이려면 아트 교체(배경 1장 + 컷아웃 캐릭터 2장)가 선행돼야 합니다.

→ **결정 필요**: 아트를 새로 뽑을 것인가? 아니면 현재 타협선에서 폴리시만 할 것인가?

## 반드시 알아야 할 함정 (여기서 시간 다 씀)

### 1. 추측하지 말고 측정할 것
레이아웃이 깨지면 `get_combined_minimum_size().y`를 찍어보세요. **5분이면 범인이 나옵니다.**
이번 세션에서 두 번 헛짚었고(초상화 탓/카드 크기 탓), 측정하자마자 진범이 나왔습니다.

```gdscript
for path in ["BattleField", "BattleField/VBoxLayout", ...]:
    var n: Control = get_node_or_null(path)
    if n: print("%s size.y=%.1f min.y=%.1f pos.y=%.1f" % [path, n.size.y,
        n.get_combined_minimum_size().y, n.global_position.y])
```

### 2. 초상화는 범인이 아니다
`PlayerFaceTexture`/`BossFaceTexture`는 `expand_mode = 3`(IGNORE_SIZE)이라 **최소 높이가 0**.
아무리 줄여도 공간이 안 나옵니다. 진짜 바닥은 `BossInfoRight`(384) / `PlayerAbility`(301) / `StatusBars`(98).

### 3. 모듈 슬롯(152×212)은 줄일 수 없다
슬롯이 `PanelContainer`라 자식을 **강제 리사이즈**하는데, 장착 카드(146×206)를 `size`로 줄이면 **글자가 넘칩니다**.
줄이려면 카드에 `scale`을 걸어야 합니다 (마켓 썸네일이 그 방식 — 지금은 revert됨).

### 4. 카드 축소는 `size`가 아니라 `scale`
`size`로 줄이면 앵커된 자식(글자)이 안 줄어서 넘칩니다. `scale`은 폰트까지 함께 축소되고, MSDF라 선명하게 유지됩니다.

### 5. 드로우 순서 = 선언 순서
형제 노드는 **나중 선언이 위에 그려짐**. 무대(`BossArea`)가 상단바를 덮어서 `MiddleWrapper`에 `z_index = 10`을 줬습니다.
"상단바 소실" 버그가 두 번 났는데 **원인이 완전히 달랐습니다**:
- 1차: `MiddleWrapper` 최소높이 0 → 공간 부족 시 붕괴 → `custom_minimum_size = (0, 52)`
- 2차: 무대가 위에 그려짐 → `z_index`

### 6. 앵커 Control도 size가 min으로 클램프된다
앵커로 준 높이가 최소 크기보다 작으면 무시되고 넘칩니다.

### 7. `%UniqueName`은 재부모화해도 안 깨진다
스크립트가 `%`로 노드를 잡으니 트리를 옮겨도 로직은 살아있습니다.
**단 하드코딩 경로**(`get_node_or_null("BattleField/VBoxLayout/...")`)는 같이 고쳐야 합니다 — `_apply_premium_ui`, `_setup_game_log`가 그런 케이스.
