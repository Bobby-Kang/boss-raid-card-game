## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 1:1 보스 레이드 카드게임 (턴제) |
| 플레이어 수 | 1명 (vs AI 보스) |
| 핵심 메카닉 | 덱 비셔플 / AP 자원 / 라운드 턴 오더 |
| 레퍼런스 | Aeon's End (덱 순서 보존), Slay the Spire (AP 카드 사용) |
| 엔진 | Godot 4.6.2, 1280×800 |

### 핵심 아이덴티티
> **"버린 순서가 곧 전략이다"**  
> 섞이지 않는 덱. 어떤 카드를 어떤 순서로 버리느냐가 다음 라운드를 결정한다.

## 2. 승리/패배 조건

| 조건 | 내용 |
|------|------|
| 승리 | 보스 HP를 0으로 만들기 |
| 패배 | 플레이어 HP가 0이 되거나, 미정 조건 |

---

## 3. 자원 시스템

### 3.1 AP (Action Point)
- **최대치**: 3
- **충전**: 플레이어 턴 시작 시 3으로 완전 리셋
- **소모**: 카드 사용 비용만큼 차감. AP 부족 시 카드 사용 불가
- **이월 없음**: 남은 AP는 다음 턴으로 이월되지 않음

### 3.2 골드
- **획득**: `GainGoldEffect` 카드 사용 시 증가
- **소멸**: 턴 종료 시 전량 증발 (이월 불가)
- **예비 슬롯 예외**: 골드 카드를 예비 슬롯에 보관하면 사용하지 않고 다음 턴으로 이월 가능
- **용도**: 현재 미사용 (향후 마켓 구매에 활용 예정)

---

## 4. 덱 시스템

### 4.1 비셔플 덱 (Aeon's End 방식)
```
뽑을 더미 → 손패(5장) → 버린 더미
               ↑                  ↓
         (뽑을 더미 소진 시)  (역순 재적재)
```
- 버린 더미가 역순으로 뽑을 더미가 됨
- **전략적 함의**: 마지막에 버린 카드가 가장 먼저 다시 뽑힘

### 4.2 손패 규칙
- 턴 시작 시 5장 드로우
- 턴 종료 시 손패 전체를 버린더미로 이동 (순서 선택 UI)
- 카드 사용 또는 버리기 외에 손패에 남길 수 없음

## 5. 카드 시스템

### 5.1 카드 타입

| 타입 | 설명 | 현재 구현 |
|------|------|----------|
| ATTACK | 피해 공격 카드 | ✅ |
| SKILL | 방어·유틸리티 카드 | ✅ |
| POWER | 지속 효과 카드 (미구현) | 계획 중 |
| MODULE | 액티브 슬롯에 장착하는 영구 강화 카드 | ✅ (슬롯만) |

### 5.2 카드 효과 클래스

| 클래스 | 효과 |
|--------|------|
| `DamageEffect(amount)` | 보스에게 `amount` 피해 |
| `BlockEffect(amount)` | 플레이어 방어력 +`amount` |
| `GainGoldEffect(amount)` | 골드 +`amount` |
| `DrawEffect(amount)` | 카드 `amount`장 즉시 드로우 |
| `DiscardEffect(amount)` | 손패에서 `amount`장 선택 버리기 |

### 5.3 스타터 덱 (10장)

| 카드명 | 타입 | AP 비용 | 효과 | 수량 |
|--------|------|---------|------|------|
| 골드 카드 | SKILL | 0 | 골드 +1 | 5 |
| 베기 | ATTACK | 1 | 보스 피해 2 | 2 |
| 막기 | SKILL | 1 | 방어력 +3 | 2 |
| 집중 | SKILL | 0 | 카드 드로우 1 | 1 |

## 6. 전투 수치

### 6.1 플레이어

| 항목 | 수치 |
|------|------|
| 최대 HP | 50 |
| 시작 HP | 50 |

### 6.2 보스 (현재: 기본 보스)

| 항목 | 수치 |
|------|------|
| 최대 HP | 100 |
| 시작 HP | 100 |
| 기본 공격 | 매 보스 턴 플레이어에게 5 피해 |

### 6.3 방어력 (Block)
- 피해를 먼저 흡수. 방어력이 피해보다 적으면 초과분만 HP 감소
- **플레이어 방어력**: 다음 플레이어 턴 시작 시 리셋
- **보스 방어력**: 다음 보스 턴 시작 시 리셋

---

## 7. 라운드/턴 시스템

### 7.1 라운드 구조
- **5턴/라운드**
- **턴1**: 항상 플레이어
- **턴2~5**: `[플레이어, 플레이어, 보스, 보스]` 4장을 랜덤 셔플

### 7.2 턴 오더 제약
- 보스 3연속 금지 (턴1 포함하여 검증)
- 유효한 배치가 나올 때까지 최대 100회 셔플 재시도

### 7.3 플레이어 턴 흐름
1. 예비 슬롯 카드 손패 복귀
2. AP 3으로 리셋
3. 페이즈 배너 표시
4. 카드 5장 드로우
5. 카드 드래그&드롭으로 사용/버리기
6. 턴 종료 버튼 → 버린 순서 선택 → 다음 턴

### 7.4 보스 턴 흐름
1. 페이즈 배너 표시
2. 고정 패턴 실행 (현재: 5 피해)
3. 자동으로 다음 턴 진행

## 8. 슬롯 시스템

### 8.1 예비 슬롯 (Reserve Slot)
- **용량**: 1장
- **허용 카드**: `GainGoldEffect`를 가진 카드 (골드 카드)
- **동작**: 드래그&드롭으로 보관 → 다음 플레이어 턴 시작 시 손패로 복귀
- **목적**: 골드를 낭비하지 않고 적절한 타이밍에 활용

### 8.2 액티브 슬롯 (Active Slot)
- **용량**: 2슬롯
- **허용 카드**: `CardType.MODULE`
- **동작**: 장착 → 영구 효과 부여 (미구현). 교체 시 기존 모듈 손패로 복귀
- **목적**: 향후 패시브 강화 시스템의 기반

## 9. UI 구조

### 9.1 화면 레이아웃 (1280×800)
```
┌─────────────────────────────────────────┐
│              BossArea                   │  ← 보스 HP·방어력
├──────────────┬──────────────────────────┤
│ PlayerAbility│      MiddleArea          │  ← 슬롯 / 턴 오더·마켓·로그
├──────────────┴──────────────────────────┤
│              BottomArea                 │  ← 플레이어 HP·버튼·손패·더미
└─────────────────────────────────────────┘
```

### 9.2 MiddleArea 패널
| 패널 | 현재 상태 | 내용 |
|------|----------|------|
| TurnInfoPanel | ✅ 구현 | 라운드 번호, 5턴 오더 표시 |
| MarketPanel | UI만 | 라운드 마켓 (미구현) |
| ActionLogPanel | UI만 | 행동 로그 (미구현) |

### 9.3 드롭 존
| 존 | 위치 | 허용 카드 |
|----|------|----------|
| PlayDropZone | MiddleArea 오버레이 | 손패 전체 |
| DiscardPile | 버린더미 영역 | 손패 전체 |
| ReserveSlot | PlayerAbility | 골드 카드 |
| ActiveSlot1·2 | PlayerAbility | MODULE 카드 |
