## 3. 자원 시스템

### 3.1 AP (Action Point)
- **최대치**: 3
- **충전**: 플레이어 턴 시작 시 3으로 완전 리셋
- **소모**: 카드 사용 시 비용만큼 차감. AP 부족 시 사용 불가
- **이월 없음**: 남은 AP는 다음 턴으로 넘어가지 않음 — 대신 턴 종료 시 **투기 스택으로 치환** (부수 공급)

### 3.1.1 투기 (전사 고유 자원)
- **주 공급**: **ATTACK 카드 사용 시 +1** — *휘두를수록 달아오른다*
- **부수 공급**: 턴 종료 시 남은 AP 치환
- **상한 10**: 도달 시 `투기 발산` 버튼 활성화 → 보스 10 피해 + 방어도 +10, 스택 0으로
- **능동 소비**: *분노의 직격*(투기 5 소비) 등 콤보 카드가 스택을 자원처럼 사용

### 3.2 골드
- **획득**: `GainGoldEffect` 카드 사용 시 증가
- **소멸**: 턴 종료 시 전량 증발
- **용도**: 라운드 마켓에서 카드 구매 (각 카드별 고정 가격) + 마켓 리롤(3 골드)

## 4. 타임라인 파이프 시스템

### 4.1 핵심 구조
```
[타임라인 파이프 / queue_cards[]]
  1번 카드 → 2번 카드 → 3번 카드 → ... → N번 카드
        ↓ (턴 시작, 앞 5장 드로우)
[손패 존 / hand_cards[]]
  카드A  카드B  카드C  카드D  카드E   ← 사용 가능
        ↓ (사용 or 버리기)
[타임라인 파이프 맨 뒤로 추가]
```

- `queue_cards[]`: 파이프 대기 중인 카드 (번호 순으로 표시)
- `hand_cards[]`: 현재 손패 (최대 5장, 드래그 조작 가능)
- 카드를 사용하거나 버리면 → 파이프 맨 뒤(Tail)로 추가
- 턴 시작 시 파이프 앞에서 5장 꺼내 손패로

### 4.2 FIFO 순환 원리
- **게임 시작 시 스타터 덱을 1회 셔플** → 매 판 다른 시작 (첫 턴 고착화 방지)
- 이후 **게임 중에는 섞지 않음** — 버린 순서가 곧 다음에 드로우되는 순서
- 턴 종료 시 남은 손패는 **플레이어가 순서 지정** 후 파이프 맨 뒤로
- 즉 "매 판 다른 시작 + 시작 후엔 완전 예측 가능"

### 4.3 UI 구성
| 영역 | 위치 | 내용 |
|------|------|------|
| 🃏 손패 | 하단 중앙 | **부채꼴 배치**(HandBelt=Control 자유배치, 하단 중앙 회전축·카드당 4°·30px 겹침·중앙 crest) + **호버 확대 1.9×**(회전 펴짐·최상단, MSDF 폰트로 확대 시에도 선명). 바닥 정렬로 카드가 아래쪽에 안착 |
| 📜 타임라인 파이프 | 하단 우측 | 미니 카드 행(비용 칩 + 카드명 + 🔨단련 뱃지) 세로 스크롤. 행 호버 시 풀사이즈 카드 프리뷰 |

### 4.4 파이프 메커니즘 (USP 강화 — *섞지 않는 공개 FIFO*를 게임플레이로)

파이프는 *미래를 안다*(공개)는 게 유일무이한 자산. 이를 실제 결정으로 만드는 3종 메커니즘.

**① 🔨 단련 (Tempering)** — *파이프를 돌수록 강해진다*
- `card.temper`: 카드가 파이프 맨 뒤로 돌아갈 때마다 +1 (`_send_to_pipe_back`)
- `TemperedDamageEffect`: 피해 = `value + temper × per_temper`
- 파이프 행에 `🔨N` 뱃지. *지금 쓸지 / 한 바퀴 더 묵혀 키울지* 결정
- 적용: 단련된 참격(6+3/회), 혼신의 일격(5+2/회)

**② 🔀 조작 (Manipulation)** — *미래를 다시 쓴다*
- `reorder_pipe_to_front`: 파이프 카드 1장 선택 → 맨 앞으로 (`ReorderPipeEffect`)
- `rewind_pipe`: 맨 뒤 N장을 맨 앞으로, 방금 버린 카드 회수 (`RewindPipeEffect`)
- 적용: 운명 재배치(선택 1장), 기세 몰이(맨 뒤 1장 회수)

**③ 🔗 인접 (Adjacency)** — *순서가 곧 콤보*
- `pipe_front_has_type(type, count)`: 파이프 맨 앞 N장에 특정 타입이 있으면 보너스
- `AdjacencyDamageEffect` / `AdjacencyBlockEffect`: 조건 충족 시 추가 피해/방어
- 적용: 돌격(다음이 공격이면 +4), 굳건한 방패(다음이 스킬이면 +3)

**④ 🚩 선봉 (Vanguard)** — *미래 손패의 선두에 세워라* (타임라인 전쟁 업데이트)
- `card.vanguard`: 이번 손패의 **첫 장**(파이프 맨 앞이었던 카드)으로 드로우되면 true (`_draw_hand`에서 i==0만)
- `VanguardDamageEffect` / `VanguardBlockEffect`: 선봉이면 보너스
- 턴 종료 순서 지정이 직접 보상으로 연결 — "이 카드를 미래 손패 1번에 놓겠다"는 계산 플레이
- 적용: 선봉 돌파(4+6), 선봉 방패(4+5)

**⑤ ⏮ 보스 타임라인 간섭** — *보스의 미래도 조작한다* (타임라인 전쟁 업데이트)
- `BossDeckSystem` 간섭 API 3종: `push_front_to_back()`(밀기) / `delay_all_powers(n)`(지연) / `break_ready_power(min_tokens)`(파괴 — 카운트 2+만, 임박한 파워는 불가)
- 카드 4장: **밀어내기**(다음 예고를 뒤로 3장 — 삭제 아닌 지연), **시간의 족쇄**(POWER 카운트 +1 — 야수의 외침 미러), **파워 브레이커**(카운트 2+ POWER 파괴), **예지의 일격**(보스 다음이 공격이면 +6 — 보스 타임라인 읽기)
- `ctx.boss_next_card_is_attack()` 헬퍼

**파이프 UI — 미래 손패 구분선**: 파이프 목록에 5장(DRAW_COUNT)마다 `▼ 다음 손패 / ▼ N턴 뒤 손패` 라벨 삽입 — 순서 지정이 "몇 턴 뒤 어느 손패를 조립하는지" 눈에 보임. (드로우 봉인 시 경계가 어긋날 수 있음 — 계획 보조 용도)

**구현 메모**: 효과는 `execute(ctx)` 시그니처 유지. main_scene이 효과 실행 직전 `ctx.acting_card`를 세팅하고 파이프 조작 Callable 3종(`peek_pipe_front`/`reorder_pipe_to_front`/`rewind_pipe`)을 등록 → 효과가 ctx 경유로 자기 단련도·파이프 상태를 조회. 효과는 **파이프로 보내기 전**에 실행되어 인접 판정이 정확.

## 5. 카드 시스템

### 5.1 카드 타입

| 타입 | 설명 | 구현 상태 |
|------|------|----------|
| ATTACK | 피해 공격 카드 | ✅ |
| SKILL | 방어·유틸리티 카드 | ✅ |
| POWER | 지속 효과 카드 | 계획 중 |
| MODULE | 액티브 슬롯에 장착하는 영구 강화 카드 | ✅ (반격 태세 구현) |

### 5.2 카드 효과 클래스

| 클래스 | 효과 |
|--------|------|
| `DamageEffect(amount)` | 보스에게 `amount` 피해 |
| `BlockEffect(amount)` | 플레이어 방어력 +`amount` |
| `BlockDamageEffect` | 현재 플레이어 방어도만큼 보스에게 피해 (value 미사용) |
| `GainGoldEffect(amount)` | 골드 +`amount` |
| `DrawEffect(amount)` | 파이프 앞에서 `amount`장 즉시 손패로 |
| `ExileEffect(amount)` | 손패에서 `amount`장 선택해 **영구 소멸** (파이프 미복귀) |
| `RemoveCardEffect` | 손패 또는 파이프에서 1장 영구 제거 (선택 UI 통해) |

각 효과는 호버 프리뷰용 메타데이터를 `get_preview_summary() -> Dictionary` 가상 메서드로 노출. UI는 효과 클래스 이름·경로에 의존하지 않고 이 메서드 결과만 사용.

### 5.3 카드 키워드

| 키워드 | 효과 |
|--------|------|
| `consume = true` | 사용 후 파이프로 돌아가지 않고 **영구 소멸** (예: 마켓 *각인의 의식*) |

### 5.4 스타터 덱 (10장)

> 초반부터 전투 가능하도록 골드 5→3 축소. 스타터는 **순수 기본기**(공격·방어·드로우·자원)만 — 콤보(인접 등)는 마켓에서 배움. 시작 시 파이프를 **1회 셔플**하므로 매 판 첫 손패가 달라져 첫 턴 고착화가 없다 (게임 중엔 섞지 않음 — 파이프 컨셉 유지).

| 카드명 | 타입 | AP 비용 | 효과 | 수량 |
|--------|------|---------|------|------|
| 골드 카드 | SKILL | 1 | 골드 +3 | 3 |
| 베기 | ATTACK | 1 | 보스 피해 3 | 4 |
| 막기 | SKILL | 1 | 방어력 +3 | 2 |
| 집중 | SKILL | 0 | 카드 드로우 1 | 1 |

## 6. 전투 수치

### 6.1 플레이어 (Warrior)

| 항목 | 수치 |
|------|------|
| 최대 HP | 60 |
| 시작 HP | 60 |
| AP | 3 (매 플레이어 턴 리셋) |

### 6.2 보스 (기본 보스)

| 항목 | 수치 |
|------|------|
| 최대 HP | 80 |
| 시작 HP | 80 |
| 공격 | 카드 기반 (Phase별 다양, BGD 7.6 참조) |
| 페이즈 | 3단계 (HP 임계 **또는** 덱 진입 트리거, 단방향) |

### 6.4 보스 페이즈

| 페이즈 | HP 임계 (boss_max_hp 비율) | 마켓 가중치 (T1/T2/T3) |
|--------|---------------------------|------------------------|
| 1 | — (초기) | 100 / 0 / 0 |
| 2 | ≤ 66% | 20 / 100 / 0 |
| 3 | ≤ 33% | 5 / 30 / 100 |

- **트리거**: HP 임계 도달 시 발화 (`_on_boss_hp_changed` → `check_hp_trigger()`)
- **단방향**: 페이즈는 올라가기만 함 (HP 회복 시 내려가지 않음)
- **마켓 영향**: 페이즈가 바뀐 다음 `refresh_slots()`부터 가중치가 반영됨

### 6.3 방어력 (Block)
- 피해를 먼저 흡수. 초과분만 HP 감소
- **플레이어**: **라운드 종료 시 리셋** (한 라운드 내 4턴 동안 누적/유지)
- **보스**: **보스 턴 시작 시 리셋** — 웅크리기·강철 벽 방어는 다음 보스 턴까지만 유지 (잔존 시 영구 벽이 되던 문제 수정)

## 7. 라운드/턴 시스템

### 7.1 라운드 구조
- **4턴/라운드**
- **고정 교대**: 플레이어 → 보스 → 플레이어 → 보스 (셔플 없음)
- 예측 가능한 순서로 전략적 계획을 가능하게 함

### 7.2 턴 오더 UI
- 라운드 시작 시 4개 슬롯 모두 처음부터 공개 표시
- 현재 턴: 밝게 강조 / 지난 턴: 40% 투명 / 이후 턴: 70% 투명

### 7.3 플레이어 턴 흐름
1. AP 3으로 리셋
2. 장착된 모듈 훅 실행 (`on_player_turn_start`) — 예: 견고한 갑옷 → 방어도 +2
3. 페이즈 배너 표시
4. 파이프 앞 5장 → 손패로 드로우
5. 손패에서 드래그&드롭으로 카드 사용/버리기
6. 턴 종료 버튼 → 남은 손패 순서 지정 → 파이프 맨 뒤로

### 7.4 보스 턴 흐름
1. 파워 카드 카운트다운 틱 — 0이 된 파워 카드 즉시 발동 후 버린 카드 더미로
2. 덱 앞에서 카드 1장 드로우
3. 턴 인디케이터·보스 인텐트 갱신
4. **보스 행동 카드 연출** (`BossActionPresenter`) — 텍스트 배너 대신 실제 카드를 화면 중앙에 큰 카드로 제시
   - **ATTACK**: 보스 쪽에서 날아와 제시 → 플레이어 쪽으로 **슬램 + 임팩트(셰이크/플래시)** → 효과 적용(`resolve_cb`로 임팩트 순간 동기화) → 버린 카드 더미
   - **POWER**: 제시 후 **파워 존으로 흡수**되듯 축소 이동 → 파워 존 배치(카운트다운 토큰) → 이후 매 보스 턴 -1 → 0 시 발동
   - **발동된 파워(TRIGGER)**: "💥 파워 발동!" 헤더로 카드 슬램 연출
   - **무효화(NEGATED)**: *불멸의 방패* 등으로 막히면 "🛡 행동 무효화!" + 카드 흔들림 후 폐기
   - **카드별 시그니처 VFX** (`_play_card_signature_fx`): ATTACK/TRIGGER 임팩트 순간, 카드명별로 고유 이펙트가 얹힌다. 재사용 프리미티브(`_spawn_ring` 확장/수축 링, `_spawn_streak` 회전 섬광선)를 조합한 **7종 아키타입** — slash(발톱 3연격)/impact(충격 링+힛스톱)/roar(동심원 충격파)/poison(초록 독무 링)/shield(방어막 수축 링)/charge(수평 돌진 스트릭)/frenzy(붉은 다중 섬광 연타). `CARD_FX` 테이블이 18장 각각을 아키타입+강조색에 매핑
5. 장착된 모듈 훅 실행 (`on_boss_turn_end`)
6. 자동으로 다음 턴 진행

> 연출 흐름은 `await` 기반 — 카드 등장(0.35s) → 제시(0.7s) → 마무리 순으로 진행되며, 실제 피해/효과는 슬램 임팩트 순간에 적용되어 얼굴 피격 리액션과 동기화된다.

## 7.5 보스 페이즈 시스템

### 개요
보스는 전투 중 **3단계 페이즈**를 거친다. 페이즈가 올라갈수록 마켓 카드 풀이 상위 티어로 이동한다.

### 전환 트리거 (둘 중 **먼저 오는 쪽** = OR, 단방향)
- **① HP 임계 도달** — 보스 HP 변경 시 `phase_system.check_hp_trigger()`
- **② 보스 덱 진입** — 보스가 상위 페이즈 카드를 뽑는 순간 `phase_system.check_deck_trigger(get_phase_of(card))` (보스 행동군과 게임 페이즈의 엇박 제거)
- 단방향 — 페이즈는 올라가기만 함 (HP 회복 시 내려가지 않음)

| 페이즈 | HP 임계 |
|--------|---------|
| 2 진입 | ≤ 66% |
| 3 진입 | ≤ 33% |

### 평가 시점
- 보스 HP 변경 시 `_on_boss_hp_changed` → `check_hp_trigger()`
- 보스 카드 드로우 직후(`_begin_boss_turn`) → `check_deck_trigger()`
- 공통 승격 로직은 `_advance_to(target)` — target > 현재일 때만 `phase_changed` emit

### UI / 연출 (순수 연출 — 능력치 변화 없음)
- `%PhaseLabel` 텍스트 "페이즈 N" + 색상(흰 → 청 → 빨)
- **페이즈 전환 시네마틱** (`_play_phase_transition_fx`): 페이즈 색 화면 섬광 + 강한 화면 흔들림 + 보스 얼굴 "강해졌다" 포효(확대 + **붉은 기운 2연타** 펄스) + 큰 컬러 타이틀(`phase_banner.show_title`, `⚔ PHASE N ⚔` + 부제 + 스케일 팝)
- **각성 능력치 없음** — 전환 시 보스에게 방어막·공격 버프를 주는 실제 강화는 넣지 않음(스노볼로 난이도 스파이크 → 폐기). 플레이어에겐 오히려 **카드 제거 보상**이 주어짐

### 구현
- `scripts/bosses/boss_phase_system.gd` — `BossPhaseSystem` (RefCounted, `GameContext` 주입)
- `GameContext`와 분리: 페이즈는 **보스 전용 상태**이므로 공유 컨텍스트에 추가하지 않음
- 시그널: `phase_changed(new_phase, old_phase)` → `MarketPanel.set_phase()`·UI 갱신

## 7.6 버그베어 카드 풀 (18장) — 페이즈 정체성 재설계

> *"피 냄새 맡은 야수 — 추적 → 광폭화 → 광기의 결정타."*

각 페이즈에 시그니처 메커니즘 부여. POWER는 *단일 효과*(카운트 0 시 발동)로 단순화 (다단 효과 폐지 — 인지 부하 절감).

### Phase 1 (6장) — 🐺 추적자 — *디버프 깔기·시그니처 POWER 1장*

ATTACK 5장이 취약·드로우 봉인을 매턴 깔고, POWER는 **사냥의 시작** 단 1장 — 페이즈 클라이맥스 한 방.

| 카드 | 타입 | 카운트 | 효과 |
|---|---|---|---|
| 할퀴기 | ATTACK | — | 플레이어 4 피해 |
| 맹독 발톱 | ATTACK | — | 3 피해 + 취약 1 |
| **위협** | ATTACK | — | 3 피해 + 드로우 봉인 1 |
| **으르렁** | ATTACK | — | 4 피해 + 취약 1 |
| **웅크리기** | ATTACK | — | 보스 방어 +5 + 방어도만큼 반사 피해 |
| **사냥의 시작** ⭐ | POWER | 4 | 💥 **피 냄새 활성** + 10 피해 + **취약 3** |

### Phase 2 (6장) — 💢 광폭화 — *즉발 강타·시그니처 POWER 1장*

ATTACK 5장 (강타·연속 할퀴기·잔혹한 일격·야수의 외침·분노의 포효) + POWER 1장(강철 벽). 보스가 매턴 직접 때리는 페이즈. 자기 강화(공격력 +2)·회복·POWER 가속이 즉발 ATK로 들어와 빠른 압박.

| 카드 | 타입 | 카운트 | 효과 |
|---|---|---|---|
| 강타 | ATTACK | — | 11 피해 |
| 연속 할퀴기 | ATTACK | — | 4 피해 × 3회 (공격력 보너스 매회 자동 적용) |
| **잔혹한 일격** | ATTACK | — | 8 피해 + 취약 2 + 보스 HP +2 회복 |
| **야수의 외침** | ATTACK | — | 모든 활성 POWER 카운트 -1 + 보스 방어 +4 |
| **분노의 포효** | ATTACK | — | 보스 공격력 +2 영구 |
| **강철 벽** ⭐ | POWER | 2 | 💥 보스 방어 **+15** + **HP +6 회복** |

### Phase 3 (6장) — 💀 광기 — *큰 즉발 ATTACK 5장 + 결정타 POWER 1장*

ATTACK 5장이 평균 8~11 피해. POWER는 **최후의 발악**(C3, **28 피해**) 단 1장 — 죽거나 막거나.

| 카드 | 타입 | 카운트 | 효과 |
|---|---|---|---|
| 광기의 돌진 | ATTACK | — | 11 피해 |
| 광란 | ATTACK | — | 10 피해 + 보스 자해 3 |
| 분쇄 | ATTACK | — | 8 피해 + 드로우 봉인 2 |
| 피의 추격 | ATTACK | — | 플레이어 HP ≤30% → **15 피해**, 아니면 5 피해 |
| **절망의 포효** | ATTACK | — | 8 피해 + 드로우 봉인 2 |
| **최후의 발악** ⭐ | POWER | 3 | 💥 **28 피해** (결정타) |

⭐ = 시그니처 POWER — 페이즈마다 단 1장, *진짜 필살기*

### 페이즈별 정체성 요약

| 페이즈 | 시그니처 POWER | ATTACK 수 | 정체성 키워드 |
|--------|----------------|-----------|---------------|
| **P1 추적자** | 사냥의 시작 (피 냄새+10+취약3) | 5 (디버프 누적) | 취약·봉인 깔다가 *피 냄새 깨어남* 한 방 |
| **P2 광폭화** | 강철 벽 (방어 +15·회복 +10) | 5 (즉발 강타·자기 강화) | 매턴 직접 타격 → POWER로 *다시 일어섬* |
| **P3 광기** | 최후의 발악 (28 피해) | 5 (큰 즉발) | 평균 9~11 피해 ATK + 결정타 *죽거나 막거나* |

**총 POWER 3장 / ATTACK 15장** — 페이즈당 시그니처 POWER 1장만. 떴을 때 "왔다"는 무게가 진짜 필살기 수준. 파워존이 1장만 떠 있어 인지 부하 최저.

### 신규 디버프·상태 시스템

#### 플레이어가 *받는* 디버프
| 상태 | 효과 | 소진 |
|---|---|---|
| **드로우 봉인 N** | 다음 플레이어 턴 드로우 -N | 1회성 (소비 후 0 리셋) |
| **취약 N스택** | 받는 다음 N번 피해 ×1.5 | 스택당 1회 |

#### 보스 자체 상태
| 상태 | 효과 | 트리거 |
|---|---|---|
| **피 냄새** | HP ≤50% 시 모든 ATTACK ×1.2 | *사냥의 시작* 발동 후 영구 |
| **공격력 보너스** | 모든 ATTACK +N 영구 | *분노의 포효* 발동 시 |

### POWER 작동 방식

> *"POWER 카드는 카운트 0 도달 시 단일 효과 발동."*

```gdscript
@export var effects: Array[CardEffect] = []           # 카운트 0 시 발동 (또는 ATTACK 즉시)
@export var on_draw_effects: Array[CardEffect] = []   # POWER 드로우 시 즉시 (예비 — 현재 미사용)
@export var on_tick_effects: Array[CardEffect] = []   # POWER 매 보스 턴 (예비 — 현재 미사용)
```

> **다단 효과는 현재 미사용** — 인지 부하 절감을 위해 단일 효과로 통일 (2026-05-28 재설계).
> 코드/필드는 유지해 향후 시그니처 카드 1~2장에 선택 적용 가능.

`BossDeckSystem`:
- `play_card`에서 POWER 카드일 때 `on_draw_effects` 즉시 실행
- `tick_powers`에서 매 보스 턴 시작 시 `on_tick_effects` 실행 → 카운트 -1
- 카운트 0 도달 시 `effects` 실행 → 버린 카드 더미

**결과**: 깎아서 끊어도 *드로우 시 효과는 이미 발동*, 안 끊으면 *카운트 0 큰 효과* — 진짜 선택지

### 페이즈 가시화 (보스 덱 시각 분리)

덱 자체는 `[P1 셔플][P2 셔플][P3 셔플]` 순서로 쌓이지만(티어 경계 고정), 카드 인스턴스 자체에는 페이즈 정보가 없으므로 **`BossDeckSystem._card_phase` 사이드-딕셔너리**로 출처 페이즈를 추적한다.

- `get_phase_of(card) → int` — 카드의 출처 페이즈 (1/2/3)
- `get_remaining_counts_by_phase() → {1, 2, 3: n}` — 페이즈별 남은 장수
- `get_remaining_names_by_phase() → {1, 2, 3: [names]}` — 칩 툴팁용

**UI 표현:**
- **3-페이즈 칩 행** (`BossInfoRight` 상단): `P1·n / P2·n / P3·n` — 각 칩이 페이즈 색(P1 양피지·P2 청·P3 적)으로 표시되고, 소진된 페이즈는 회색 디밍. 호버 시 해당 페이즈 남은 카드 목록 툴팁. 기존 `🂠 18` 단일 라벨을 대체.
- **카드 위젯 페이즈 뱃지** (`BossCardDisplay`): 좌측 세로 컬러 스트립(6px, 페이즈 색 α0.85) + 좌상단 `P1`/`P2`/`P3` 칩. 다음 예고·보스 행동 카드 연출·파워 존 표시 모든 곳에서 카드가 어느 페이즈 소속인지 즉시 식별 가능.
- 호출 흐름: 각 `BossCardDisplay.setup(card, tokens, size, phase)`에서 phase는 `boss_deck_system.get_phase_of(card)`로 조회.

### 신규 보스 효과 클래스 (11종)

| 클래스 | 역할 |
|---|---|
| `BossDrawLockEffect` | 드로우 봉인 N 부여 |
| `BossVulnerabilityEffect` | 취약 N스택 부여 |
| `BossSelfDamageEffect` | 보스 자해 |
| `BossHealEffect` | 보스 HP 회복 |
| `BossDrainEffect` | 흡혈 (피해 + 회복) |
| `BossAttackBuffEffect` | 보스 공격력 영구 +N |
| `BossBloodScentEffect` | 피 냄새 활성화 |
| `BossPercentDamageEffect` | 플레이어 HP의 N% 피해 |
| `BossReflectDamageEffect` | 보스 방어도만큼 피해 (웅크리기) |
| `BossExecuteDamageEffect` | HP 임계 처형 (피의 추격) |
| `BossPowerSpeedupEffect` | 모든 POWER 카운트 -1 (야수의 외침) |

기존 `BossDamageEffect`는 `get_boss_attack_modifier`로 *공격력 보너스 + 피 냄새* 자동 적용.

- 배경: **픽셀 던전 무대**(§10.6) + 가장자리 비네팅 (`_setup_background_atmosphere`)
- 텍스처 필터: **전역 Linear**(`default_texture_filter=1`, 텍스트·아트 부드럽게) + **픽셀 요소만 개별 nearest**(배경·컷아웃·Kenney 픽셀 크롬 등 `texture_filter` 명시) — 전역 nearest는 폰트 안티에일리어싱을 뭉개 기각
- **폰트**: Pretendard SemiBold (OFL, `assets/fonts/`) — 테마 `default_font`. **MSDF 임포트**(`multichannel_signed_distance_field=true`)로 카드 확대 등 어떤 배율에도 글자 선명 (폰트 `.import`는 gitignore 예외로 추적)
- **창/스케일**: 디자인 해상도 1920×1080. 실행 창 `window_*_override`도 **1920×1080으로 일치**시켜 배율 1.0 확보 — 픽셀아트는 배율이 정수가 아니면 격자가 어긋난다(1600×900이면 0.833배 → 아트 3배와 곱해 2.5배가 되어 뭉갬). `2d/snap/snap_2d_transforms_to_pixel`·`snap_2d_vertices_to_pixel` 활성화 — 반 픽셀 위치에 놓인 nearest 텍스처가 흐려지는 것을 막는다(실제로 전사가 흐려졌던 원인)
  - 참고: Godot의 **자동 정수 스케일**(`stretch/scale_mode="integer"`)은 기준 뷰포트를 크게 늘릴 때만 유효 — 우리처럼 기준이 이미 1920이면 배율 0이 되어 못 쓴다

### 10.1.2 프레임 적용 범위 (`_apply_premium_ui`)
- **초상**(전사·보스): `draw_center=false` 프레임 → 아트가 깨끗이 보이고 테두리만 감쌈
- **카드**(손패·마켓·프리뷰, `Card._apply_data`): `card_frame(card_type)` 타입별 색 — 손패 카드와 타임라인 색 체계 일치
- **카드 크롬**(`_style_card_chrome`): 이름 배너 = 어두운 타입색 + 하단 타입색 라인 / 코스트 = **원형 금속 보석 뱃지**(금테+그림자) / 타입 배너 = 타입색 채움 + 아이콘(⚔공격·✦스킬·☄파워·◈모듈) / **아트 매트** = 프레임 안쪽 타입색 얇은 테두리
- **보스 카드**(`BossCardDisplay`): 붉은 프레임 + **위협 비네트**(방사형 크림슨 어둠) + 크림슨 이름/설명판 + **발광 카운트다운 뱃지**(크림슨 원형 + 발광 링 + 그림자)
- **버튼**: Kenney #001 4상태 (호버 밝아짐 / 눌림 어둡게 / 비활성 회색)
- **타임라인 파이프**: 타입색 원형 노드(타임라인 스톱) + 왼쪽 타입색 스파인 + 타입색 카드명
- **모듈 슬롯 / 미니로그 / 마켓·로그 모달**: 통일 (마켓 외곽 래퍼는 투명 처리 — 이중 프레임 방지). 모듈 슬롯은 카드 실제 크기 **152×212** + PlayerAbility 폭 stretch 0.85 — 장착 모듈이 풀사이즈로 보임
- **중앙 배틀 밴드**(BossArea): 투명 무대 → 배경 그라데이션 노출
- 드롭존 stylebox 변경이 프레임을 지우지 못하게 `DropZone`이 드래그 시작 시점의 배경을 저장·복원
- (구 `PixelFrameLayer`·`PixelFrame` — 픽셀 돌 액자 실험 코드, 현재 미사용·보존)

### 10.2.1 무대 1:1 대결 구도 (픽셀 컷아웃)
| 영역 | 내용 |
|------|------|
| `%PlayerFaceTexture` | 좌측 — 전사 **컷아웃**(투명 배경 스프라이트), 발끝을 바닥선에 맞춰 자유 배치. 피격 시 펀치+플래시 |
| `%BossFaceTexture` | 우측 — 버그베어 컷아웃 + 머리 위 인텐트 라벨 (`%BossIntentLabel`) |
| 보스 HP | **보스 머리 위 부유**(`_float_boss_hp`) — 레퍼런스처럼 적 HP가 적 위에 뜬다 |
| BossInfoRight | 가장 우측 — 페이즈/방어도/**페이즈 칩(P1·P2·P3)**/버린카드/다음예고/파워존 컬럼 |

- 초상 패널(`PlayerFace`/`BossFace` PanelContainer)과 `VsLabel`은 **숨김** — 컷아웃이 무대 배경 위에 직접 선다. 노드는 `%`로 접근하므로 재부모화해도 기존 애니메이션(포효·피격·돌진)은 그대로 동작
- ⚠️ **빈 무대 컨테이너는 `MOUSE_FILTER_IGNORE`로 둘 것** — 초상 패널을 숨기면 `TopRow`/`BossVBox`가 내용을 잃고 화면 상단까지 확장돼 상단바 버튼의 클릭을 가로챈다. **Godot의 입력 판정은 `z_index`를 무시하고 트리 순서만 본다** — `MiddleWrapper`의 `z_index=10`은 *그리기*만 고칠 뿐 입력은 무대가 먹는다

**보스 인텐트 라벨**: 보스 얼굴 상단에 다음 행동 또는 이번 턴 행동을 큰 글씨로 표시.
- 플레이어 턴: `다음 ⚔️ 카드명` (청록색)
- 보스 턴 직전: `이번 턴 ⚔️ 카드명` (오렌지색, 펄스 강조)

## 10.4 라운드 마켓

### 동작
- **상시 노출 컴팩트 스트립** (116px, BossArea와 BottomArea 사이): 모달을 없애고 항상 보이게 → 골드를 번 그 턴에 바로 구매 판단. 각 레인 = `[썸네일 44×62] [카드명 / 구매(N G)]` 가로 배치, **썸네일 호버 시 전체 카드(1.15배) 팝업**(작은 썸네일로는 효과를 못 읽으므로)
  - 썸네일은 `size`가 아닌 **transform scale**로 축소 — size로 줄이면 글자가 넘친다 (`_make_card_visual(cd, scale)`)
  - 리롤 버튼은 레인 **오른쪽 가로 배치** — 세로로 쌓으면 이 컬럼(112px)이 레인(84px)보다 높아져 마켓 전체 높이를 지배한다
  - 세로 예산: 화면 1080에서 마켓이 쓸 수 있는 건 **약 120px뿐**(§10.5 참조). 풀사이즈 카드(170px+)는 물리적으로 불가
- **라운드 시작 시**: 4개 레인 각각의 서브풀에서 **페이즈 가중치 추첨**으로 1장씩 진열 (총 4장)
- **구매**: 카드별 고정 골드 가격 소모 → 구매 즉시 **파이프 맨 뒤**에 추가. 해당 레인은 라운드 종료까지 비움
- **리롤**: 4레인 전체 동시 재추첨. 비용 — `AP 3` 또는 `골드 3` (횟수 제한 없음)
- **턴 제약**: 플레이어 턴에만 모든 마켓 버튼 활성화. 보스 턴/턴 종료 시 비활성
- **페이즈 연동**: `phase_changed` 시그널 수신 시 `set_phase(new_phase)` → 다음 `refresh_slots()`부터 가중치 적용

## 10.5 오버레이 레이아웃 (컨테이너 → 앵커 전환)

**전투 화면은 더 이상 VBoxContainer로 공간을 나누지 않는다.** 행들이 1080px를 나눠 갖는 구조는 143px 초과로 파탄났고(§12 이력), 모바일 카드게임 레퍼런스 방식(씬 = 화면 전체, HUD = 그 위 오버레이)으로 전환했다.

### 골격
```
MainScene
├── Background
├── BattleField (Control, 여백 0)          ← MarginContainer 아님
│   └── VBoxLayout (Control)               ← VBoxContainer 아님 (이름만 유지)
│       ├── MiddleWrapper   상단바  y 10~62      z_index=10
│       ├── BossArea        무대    y 0~662      ← 초상화 (396 → 662)
│       ├── BossInfoRight   보스HUD 우측 앵커
│       ├── BottomArea      하단HUD y 662~1070
│       └── EndTurnButton   우하단 118×118 원형
├── MarketWindow (모달)                     ← 상점 버튼으로 토글
└── EndTurnOverlay / PhaseBanner / GameResultScreen
```

### 규칙
- **배치는 앵커로** — 요소끼리 공간을 경쟁시키지 말 것. 겹쳐도 된다
- **드로우 순서 = 선언 순서** — 형제는 나중 선언이 위에 그려진다. 무대(BossArea)가 상단바를 덮으므로 HUD에는 `z_index` 필요. **단 입력 판정은 `z_index`를 무시**하므로 무대 컨테이너는 `MOUSE_FILTER_IGNORE`도 함께 줘야 한다
- **앵커 Control도 `size`가 최소값으로 클램프**된다 — 앵커로 준 높이가 min보다 작으면 무시되고 넘친다

### 크롬 분리 (레퍼런스 = 미니멀·부유·차가운 톤)
| 대상 | 스타일 |
|------|--------|
| 상단바 · 손패 · 자원바 | **상자 없음** — 배경 위에 요소만 뜬다 (`StyleBoxEmpty`, 여백만 유지) |
| 큰 HUD 판 (좌하단 클러스터·다음 손패) | 테두리 없는 **어두운 반투명 판**(α 0.5) |
| 작은 픽셀 크롬 (바·모듈 슬롯·아이콘 버튼) | **Kenney Pixel Adventure 타일**(§10.6) |
| 카드 | Kenney 액자 유지 — "손에 든 물건"이라 테두리가 있어야 읽힘 |

> ⚠️ **큰 면적에 액자를 두르지 말 것.** 픽셀 프레임을 큰 패널까지 씌우면 화면이 답답해져 "떠 있는 미니멀 HUD"와 정면 충돌한다. Kenney 픽셀 크롬은 **작은 요소에만** 쓴다.

### ✅ 아트 제약 해소 (2026-07-19)
과거엔 *각자 배경을 품은 사각 일러스트 2장*이라 이음새·겹침이 불가능했다. **픽셀아트 전환(§10.6)으로 배경 1장 + 투명 컷아웃 2장 구조가 되어 제약이 사라졌다** — 스프라이트는 태생이 투명 배경이라 HUD를 무대 위에 자유롭게 겹칠 수 있다.

## 10.6 픽셀아트 전환 (다크판타지)

코믹 일러스트 → **16비트 다크판타지 픽셀아트**로 아트 방향 전환. 레퍼런스 룩(배경 1장 + 컷아웃)이 구조적으로 가능해진 결정적 변화.

### 배율 규칙 (전 에셋 공통)
**아트 1px = 화면 3px.** 배경은 640×360(=1920×1080의 1/3), 캐릭터는 발끝 기준 배치. 이 규칙이 깨지면 픽셀 격자가 어긋나 즉시 티가 난다.

| 에셋 | 원본 | 화면 |
|---|---|---|
| `battle_bg.png` | 640×360 (32색) | 1920×1080 |
| `warrior_cutout.png` | 148×150 (48색) | 450px |
| `bugbear_cutout.png` | 310×170 (48색) | 510px |

- 바닥선 = 화면 **72%**(y≈778). 두 캐릭터 모두 이 선에 발을 딛고, 타원 접지 그림자를 코드로 깐다(아트에 굽지 않음 — 그래야 위치·크기를 자유롭게 조절)
- 배치는 `_setup_pixel_stage()` / `_place_fighter()` — 컨테이너에서 빼내 자유 앵커로. **위치는 반드시 정수로 반올림**(반 픽셀이면 nearest가 뭉갠다)

### 생성·후처리 파이프라인 (`tools/pixelize.py`)
Gemini는 "픽셀아트처럼 보이는 그림"만 낸다(픽셀 크기 제각각 + 안티에일리어싱). **프롬프트로는 못 고치고 축소로 격자를 강제한다.**

```
Gemini 생성(마젠타 배경) → 키아웃 → 고립 조각 제거 → 알파 침식 → BOX 축소 → 팔레트 양자화
```

**함정 (전부 실제로 당함)**
- **양자화 방식이 콘텐츠마다 다르다**: 배경 = `maxcov`(medcut은 화면 1%인 횃불을 팔레트에서 버려 **광원이 꺼진다**) / 캐릭터 = `medcut`(maxcov는 얼굴 살색을 분홍 얼룩으로 뭉갠다)
- **`--tol`을 200 넘기지 말 것** — 마젠타-살색 거리가 ~215라 **얼굴이 통째로 지워진다**(구멍을 "노이즈"로 오진했던 버그)
- **Gemini가 4점 별 로고를 그림에 구워 넣는다** — 배경은 톤 맞는 패치로 덮고, 캐릭터는 `--min-part`(고립 조각 제거)로 자동 처리
- **얼굴 선명도는 크기에서 나온다** — 색 수를 줄여도 거의 개선 안 됨. 전사는 native 150 이상이어야 얼굴이 읽힌다
- 새 에셋은 **Godot 임포트 필요**: `Godot_console.exe --headless --path . --import` (`run_project`는 임포트를 안 한다)

### idle 호흡 애니메이션
`tools/make_idle.py` — 원본 스프라이트 **한 장에서** 상체만 1px 올려 프레임을 합성한다(`_idle/`). 엔진은 `1→2→1→3`을 0.55초 간격으로 순환(`_start_idle_breathing`).

> ⚠️ **AI로 애니메이션 프레임을 생성하지 말 것.** 실측 결과 정합(좌우·발끝)은 완벽했으나 매 프레임 그림을 다시 그려 **다리가 37% 변했다**(호흡이면 0%여야 함) — 애니메이션으로 돌리면 호흡이 아니라 지글거림으로 보인다. 합성 프레임은 가슴 44%·다리 0%로 정상.
> 단, 허리에서 **딱 잘라** 올리면 방패·검처럼 경계를 가로지르는 물체가 갈라진다. 공격·피격 등 본격 모션은 합성으로 불가 → 애니메이션 포함 에셋 팩 검토 필요(§11 Phase F).

### Kenney 픽셀 UI 크롬
**UI Pack · Pixel Adventure (CC0)** — `assets/art/Kenny/pixel_adventure/`. 원본 16/32px 타일을 **정수배로 구워** `_baked/`에 저장(그대로 쓰면 UI 픽셀이 무대보다 촘촘해 격자가 어긋난다). 속은 어두운 반투명으로 치환해 무대가 비치게 유지 — 테두리만 Kenney.

| 용도 | 소스 | 구운 배율 |
|---|---|---|
| HP 바 트랙 | 작은 타일 129-131 | ×2 → 96×32 (붉은 프레임 → 청회색 리컬러) |
| 원형 아이콘 버튼 | 큰 타일 41 | ×2 → 64×64 |
| 모듈 슬롯 | 작은 타일 58 | ×3 |

> Kenney 팩에는 **바/슬라이더 전용 에셋이 없다**(Border·Divider·Panel뿐). 위 타일은 진행바 프레임으로 전용한 것. 얇은 바에 9-slice를 쓰면 여백이 높이를 초과해 깨진다는 과거 기록도 유효.

## 10.7 ESC 메뉴 (타이틀 화면 대체)

타이틀·캐릭터 선택 화면을 **삭제**하고 실행 즉시 전투로 진입한다(캐릭터 선택은 전투 씬에 아무것도 넘기지 않아 안전했음). 대신 전투 중 **ESC**로 메뉴를 연다.

| 항목 | 동작 |
|---|---|
| ▶ 계속하기 | 메뉴 닫기 |
| ↻ 다시 시작 | `reload_current_scene()` |
| 📖 도움말 | 게임 방법 (`HelpText.BBCODE`) |
| ✕ 종료 | `get_tree().quit()` |

- ESC 재입력 시 닫힘. 도움말이 열려 있으면 **도움말이 먼저** 닫힌다
- **`get_tree().paused`는 쓰지 않는다** — 트리를 멈추면 메뉴 버튼 자체도 처리를 멈춰 클릭이 안 먹는다. 턴제라 실제 정지가 불필요하고, 딤 오버레이(ColorRect)가 뒤쪽 입력을 막는다
- 도움말 본문은 구 타이틀 화면에서 `scripts/ui/help_text.gd`(`class_name HelpText`)로 **보존·이전**
- 승패 화면의 "다시 하기"도 타이틀 복귀 → **전투 재시작**으로 변경(삭제된 씬을 참조해 크래시할 뻔한 것)

- 화면 상단 `%TurnIndicatorLabel` + `%TurnHintLabel`

- 보스 턴: `🔴 보스 턴` (빨강) + 힌트 `보스 행동: {카드명}`

- **모듈 발동 기록** (`_run_module_hooks` + `_log_module_result`): 모듈은 카드 사용 로그가 없어 턴 사이에 조용히 발동 → 반격 태세 피해 등이 기록에 안 잡히던 문제. 훅 호출을 공용 헬퍼로 감싸 **전후 실측 델타**(보스 HP·플레이어 방어도·플레이어 HP)를 계산해 `◈ {모듈명} ⇒ 보스 N 피해, 방어 +N` 형식으로 기록. 변화가 없으면(조건 미충족) 로그를 남기지 않아 소음 없음. 모듈별 로그 코드가 필요 없어 신규 모듈도 자동 커버
- **A. 전체 로그 모달**: 상단바 `📜 기록` 버튼 토글 → 중앙 오버레이(마켓 모달 패턴). 스크롤 + 열 때 최신으로 자동 스크롤. 딤/✕ 닫기
- **B. 미니 로그**: 타임라인 파이프 오른쪽 패널에 최근 6줄 상시 표시 (하단 정렬)

## 10.6 보상형 카드 영구 제거

자동 소멸 시스템 폐지 → 빌드 게임 본질에 맞는 *보상형 도구*로 전환.

### 두 트리거
1. **페이즈 전환 보상** — 보스 HP가 66% / 33% 임계 돌파 시 1회씩 자동 발생
   - 배너: `🌟 전사의 깨달음 — 보스의 약점이 보인다` 안내 후 선택 오버레이
   - 페이즈 전환 시그널 콜백 도중 await race를 피하기 위해 **`_pending_rewards` 큐**에 enqueue → `_begin_player_turn` 시작 시 안전하게 처리
2. **마켓 *각인의 의식*** (`market_engraving.tres`, 5골드, T2 특수 레인)
   - 1 AP 사용 → 손패/파이프 카드 1장 영구 제거
   - `consume = true` — 사용 후 카드 자체도 영구 소멸 (인플레 방지)

### 공통 흐름
- `_grant_card_removal_reward(reason_lines)` — 손패+파이프 통합 선택 UI
- 1장 선택 또는 **스킵 가능** (강제 X)
- 선택 시 `exile_animator.play(card)` — 슬라이드 → 흔들림 → 찢어짐 + "✖ 영구 소멸" + "💀 {카드명}이 영원히 사라졌다"

### 디자인 의도
- 빌드 게임에서 *카드를 잃는 것*은 *덱이 더 좋아지는 길*일 때 빛난다
- 페이즈 전환 = 게임 진행도 = 카드 제거 보상 — 서사 정합성
- 자원 통합: 골드 → 마켓 → 각인의 의식 → 빌드 정제 (새 자원 루프)
