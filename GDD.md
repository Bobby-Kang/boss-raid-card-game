# 보스 레이드 카드게임 — 게임 디자인 문서 (GDD)

> 이 문서는 게임 기획의 **단일 진실 공급원**입니다.
> 구현이 변경될 때마다 이 문서를 함께 업데이트하세요.

---

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 가제 | 프로젝트: 타임라인 |
| 장르 | 1:1 보스 레이드 카드게임 (턴제) |
| 플레이어 수 | 1명 (vs AI 보스) |
| 핵심 메카닉 | 타임라인 파이프(FIFO 큐) / AP 자원 / 라운드 턴 오더 |
| 레퍼런스 | Aeon's End (덱 순서 보존), Slay the Spire (AP 카드 사용) |
| 엔진 | Godot 4.6.2, 1920×1080 (개발용, 이후 조정 예정) |

### 핵심 아이덴티티
> **"타임라인을 설계하라"**
> 카드는 파이프를 순환한다. 쓰거나 버린 카드는 파이프 맨 뒤로 돌아온다.
> 어떤 순서로 보내느냐가 미래를 결정한다.

|------|------|
| 승리 | 보스 HP를 0으로 만들기 |
| 패배 | 플레이어 HP가 0이 됨 (추가 조건 미정) |

## 3. 자원 시스템

### 3.1 AP (Action Point)
- **최대치**: 3
- **충전**: 플레이어 턴 시작 시 3으로 완전 리셋
- **소모**: 카드 사용 시 비용만큼 차감. AP 부족 시 사용 불가
- **이월 없음**: 남은 AP는 다음 턴으로 넘어가지 않음

### 3.2 골드
- **획득**: `GainGoldEffect` 카드 사용 시 증가
- **소멸**: 턴 종료 시 전량 증발
- **용도**: 라운드 마켓에서 카드 구매 (각 카드별 고정 가격) + 마켓 리롤(3 골드)

---

## 4. 타임라인 파이프 시스템

### 4.1 핵심 구조
```
[타임라인 파이프 / queue_cards[]]
  1번 카드 → 2번 카드 → 3번 카드 → ... → N번 카드
        ↓ (턴 시작, 앞 5장 드로우)
[손패 존 / hand_cards[]]
  카드A  카드B  카드C  카드D  카드E   ← 사용 가능
        ↓ (사용 or 버리기)
[타임라인 파이프 맨 뒤로 추가]
```

- `queue_cards[]`: 파이프 대기 중인 카드 (번호 순으로 표시)
- `hand_cards[]`: 현재 손패 (최대 5장, 드래그 조작 가능)
- 카드를 사용하거나 버리면 → 파이프 맨 뒤(Tail)로 추가
- 턴 시작 시 파이프 앞에서 5장 꺼내 손패로

### 4.2 FIFO 순환 원리
- 덱을 **섞지 않음**
- 버린 순서가 곧 다음에 드로우되는 순서
- 턴 종료 시 남은 손패는 **플레이어가 순서 지정** 후 파이프 맨 뒤로

### 4.3 UI 구성
| 영역 | 위치 | 내용 |
|------|------|------|
| 손패 존 (ACTIVE ZONE) | 하단 중앙 | 파란 테두리, 드래그 가능한 손패 5장 |
| 타임라인 파이프 | 하단 우측 | 번호(노란색) + 카드명 + 비용 목록, 세로 스크롤 |

## 5. 카드 시스템

### 5.1 카드 타입

| 타입 | 설명 | 구현 상태 |
|------|------|----------|
| ATTACK | 피해 공격 카드 | ✅ |
| SKILL | 방어·유틸리티 카드 | ✅ |
| POWER | 지속 효과 카드 | 계획 중 |
| MODULE | 액티브 슬롯에 장착하는 영구 강화 카드 | ✅ (반격 태세 구현) |

### 5.2 카드 효과 클래스

| 클래스 | 효과 |
|--------|------|
| `DamageEffect(amount)` | 보스에게 `amount` 피해 |
| `BlockEffect(amount)` | 플레이어 방어력 +`amount` |
| `GainGoldEffect(amount)` | 골드 +`amount` |
| `DrawEffect(amount)` | 파이프 앞에서 `amount`장 즉시 손패로 |
| `DiscardEffect(amount)` | 손패에서 `amount`장 선택해 파이프 맨 뒤로 |

### 5.3 스타터 덱 (10장)

| 카드명 | 타입 | AP 비용 | 효과 | 수량 |
|--------|------|---------|------|------|
| 골드 카드 | SKILL | 0 | 골드 +1 | 5 |
| 베기 | ATTACK | 1 | 보스 피해 2 | 2 |
| 막기 | SKILL | 1 | 방어력 +3 | 2 |
| 집중 | SKILL | 0 | 카드 드로우 1 | 1 |

## 6. 전투 수치

### 6.1 플레이어 (Warrior)

| 항목 | 수치 |
|------|------|
| 최대 HP | 50 |
| 시작 HP | 50 |
| AP | 3 (매 플레이어 턴 리셋) |

### 6.2 보스 (기본 보스)

| 항목 | 수치 |
|------|------|
| 최대 HP | 100 |
| 시작 HP | 100 |
| 기본 공격 | 매 보스 턴 플레이어에게 5 피해 |

### 6.3 방어력 (Block)
- 피해를 먼저 흡수. 초과분만 HP 감소
- **플레이어**: **라운드 종료 시 리셋** (한 라운드 내 5턴 동안 누적/유지)
- **보스**: 다음 보스 턴 시작 시 리셋 (미구현)

## 7. 라운드/턴 시스템

### 7.1 라운드 구조
- **5턴/라운드**
- **턴1**: 항상 플레이어
- **턴2~5**: `[플레이어, 플레이어, 보스, 보스]` 랜덤 셔플

### 7.2 턴 오더 제약
- 보스 3연속 금지 (턴1 포함 검증)
- 유효 배치까지 최대 100회 재셔플

### 7.3 플레이어 턴 흐름
1. AP 3으로 리셋
3. 페이즈 배너 표시
4. 파이프 앞 5장 → 손패로 드로우
5. 손패에서 드래그&드롭으로 카드 사용/버리기
6. 턴 종료 버튼 → 남은 손패 순서 지정 → 파이프 맨 뒤로

### 7.4 보스 턴 흐름
1. 페이즈 배너 표시
2. 고정 패턴 실행 (현재: 5 피해)
3. 자동으로 다음 턴 진행

- **목적**: 향후 패시브 강화 시스템 기반

## 9. 카드 조작 (드래그&드롭)

| 행동 | 방법 | 결과 |
|------|------|------|
| 카드 사용 | 손패 → **가운데(PlayDropZone)** 드롭 | AP 소모, 효과 발동, 파이프 맨 뒤로 |
| 카드 버리기 | 손패 → **타임라인 파이프 패널** 드롭 | 효과 없이 파이프 맨 뒤로 |
| 모듈 장착 | 손패 → **액티브 슬롯 1·2** 드롭 | 교체 시 기존 모듈 손패 복귀 |

## 10.4 라운드 마켓

### 동작
- **라운드 시작 시**: 직업별 카드 풀에서 무작위 3장 진열
- **구매**: 카드별 고정 골드 가격 소모 → 구매 즉시 **파이프 맨 뒤**에 추가
- **리롤**: 진열 카드 3장을 모두 새로 뽑음. 비용 — `AP 3` 또는 `골드 3` (둘 다 가능, 횟수 제한 없음)
- **턴 제약**: 플레이어 턴에만 모든 마켓 버튼 활성화. 보스 턴/턴 종료 시 비활성

### 전사 카드 풀 (MVP, 4종)
| 카드명 | AP | 골드 가격 | 효과 |
|--------|----|-----------|------|
| 강타 | 2 | 4 | 보스에게 5 피해 |
| 굳건한 방패 | 1 | 3 | 방어도 +5 |
| 돌격 | 1 | 4 | 보스에게 3 피해 + 방어도 +2 |
| 명상 | 0 | 4 | 카드 2장 드로우 |

### 구현
- `scripts/ui/market_panel.gd` — `MarketPanel` (PanelContainer 상속). `card_pool: Array[CardData]` 익스포트
- 리소스: `resources/cards/warrior/market/market_*.tres` (4종)
- `CardData`에 `gold_cost: int` 필드 추가
- 시그널: `card_purchased(card_data: CardData)` → `main_scene._on_market_card_purchased`가 파이프 맨 뒤로 인스턴스화
- 트리거: `main_scene._start_round()`에서 `market_panel.refresh_slots()`, `_begin_player_turn`/`_finish_turn`에서 `set_player_turn(true/false)`

### 설계 의도
- 골드 소멸 규칙(턴 종료 시 증발)과 결합되어 **언제 골드 카드를 모아 마켓에 쓸 것인가**의 타이밍 게임이 핵심
- 리롤이 있어 진열 운에 매몰되지 않으면서, AP 3 비용으로 인해 전투 효율과의 트레이드오프 발생
- 구매 카드는 파이프 맨 뒤에 들어가므로 "지금 산 카드는 다음 라운드 후반에야 손에 들어온다" — 타임라인 설계의 일부
