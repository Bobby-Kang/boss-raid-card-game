## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 가제 | 프로젝트: 타임라인 |
| 장르 | 1:1 보스 레이드 카드게임 (턴제) |
| 플레이어 수 | 1명 (vs AI 보스) |
| 핵심 메카닉 | 타임라인 파이프(FIFO 큐) / AP 자원 / 라운드 턴 오더 |
| 레퍼런스 | Aeon's End (덱 순서 보존), Slay the Spire (AP 카드 사용) |
| 엔진 | Godot 4.6.2, 1920×1080 (개발용, 이후 조정 예정) |

### 핵심 아이덴티티
> **"타임라인을 설계하라"**
> 카드는 파이프를 순환한다. 쓰거나 버린 카드는 파이프 맨 뒤로 돌아온다.
> 어떤 순서로 보내느냐가 미래를 결정한다.

## 3. 자원 시스템

### 3.1 AP (Action Point)
- **최대치**: 3
- **충전**: 플레이어 턴 시작 시 3으로 완전 리셋
- **소모**: 카드 사용 시 비용만큼 차감. AP 부족 시 사용 불가
- **이월 없음**: 남은 AP는 다음 턴으로 넘어가지 않음

### 3.2 골드
- **획득**: `GainGoldEffect` 카드 사용 시 증가
- **소멸**: 턴 종료 시 전량 증발
- **용도**: 라운드 마켓에서 카드 구매 (각 카드별 고정 가격) + 마켓 리롤(3 골드)

## 4. 타임라인 파이프 시스템

### 4.1 핵심 구조
```
[타임라인 파이프 / queue_cards[]]
  1번 카드 → 2번 카드 → 3번 카드 → ... → N번 카드
        ↓ (턴 시작, 앞 5장 드로우)
[손패 존 / hand_cards[]]
  카드A  카드B  카드C  카드D  카드E   ← 사용 가능
        ↓ (사용 or 버리기)
[타임라인 파이프 맨 뒤로 추가]
```

- `queue_cards[]`: 파이프 대기 중인 카드 (번호 순으로 표시)
- `hand_cards[]`: 현재 손패 (최대 5장, 드래그 조작 가능)
- 카드를 사용하거나 버리면 → 파이프 맨 뒤(Tail)로 추가
- 턴 시작 시 파이프 앞에서 5장 꺼내 손패로

### 4.2 FIFO 순환 원리
- 덱을 **섞지 않음**
- 버린 순서가 곧 다음에 드로우되는 순서
- 턴 종료 시 남은 손패는 **플레이어가 순서 지정** 후 파이프 맨 뒤로

### 4.3 UI 구성
| 영역 | 위치 | 내용 |
|------|------|------|
| 손패 존 (ACTIVE ZONE) | 하단 중앙 | 파란 테두리, 드래그 가능한 손패 5장 |
| 타임라인 파이프 | 하단 우측 | 번호(노란색) + 카드명 + 비용 목록, 세로 스크롤 |

## 5. 카드 시스템

### 5.1 카드 타입

| 타입 | 설명 | 구현 상태 |
|------|------|----------|
| ATTACK | 피해 공격 카드 | ✅ |
| SKILL | 방어·유틸리티 카드 | ✅ |
| POWER | 지속 효과 카드 | 계획 중 |
| MODULE | 액티브 슬롯에 장착하는 영구 강화 카드 | ✅ (반격 태세 구현) |

### 5.2 카드 효과 클래스

| 클래스 | 효과 |
|--------|------|
| `DamageEffect(amount)` | 보스에게 `amount` 피해 |
| `BlockEffect(amount)` | 플레이어 방어력 +`amount` |
| `BlockDamageEffect` | 현재 플레이어 방어도만큼 보스에게 피해 (value 미사용) |
| `GainGoldEffect(amount)` | 골드 +`amount` |
| `DrawEffect(amount)` | 파이프 앞에서 `amount`장 즉시 손패로 |
| `DiscardEffect(amount)` | 손패에서 `amount`장 선택해 파이프 맨 뒤로 |
| `ExileEffect(amount)` | 손패에서 `amount`장 선택해 **영구 소멸** (파이프 미복귀) |

### 5.3 스타터 덱 (10장)

| 카드명 | 타입 | AP 비용 | 효과 | 수량 |
|--------|------|---------|------|------|
| 골드 카드 | SKILL | 0 | 골드 +1 | 5 |
| 베기 | ATTACK | 1 | 보스 피해 2 | 2 |
| 막기 | SKILL | 1 | 방어력 +3 | 2 |
| 집중 | SKILL | 0 | 카드 드로우 1 | 1 |

## 6. 전투 수치

### 6.1 플레이어 (Warrior)

| 항목 | 수치 |
|------|------|
| 최대 HP | 50 |
| 시작 HP | 50 |
| AP | 3 (매 플레이어 턴 리셋) |

### 6.2 보스 (기본 보스)

| 항목 | 수치 |
|------|------|
| 최대 HP | 100 |
| 시작 HP | 100 |
| 기본 공격 | 매 보스 턴 플레이어에게 5 피해 |
| 페이즈 | 3단계 (HP 임계 트리거, 단방향) |

### 6.4 보스 페이즈

| 페이즈 | HP 임계 (boss_max_hp 비율) | 마켓 가중치 (T1/T2/T3) |
|--------|---------------------------|------------------------|
| 1 | — (초기) | 100 / 0 / 0 |
| 2 | ≤ 66% | 20 / 100 / 0 |
| 3 | ≤ 33% | 5 / 30 / 100 |

- **트리거**: HP 임계 도달 시 발화 (`_on_boss_hp_changed` → `check_hp_trigger()`)
- **단방향**: 페이즈는 올라가기만 함 (HP 회복 시 내려가지 않음)
- **마켓 영향**: 페이즈가 바뀐 다음 `refresh_slots()`부터 가중치가 반영됨

### 6.3 방어력 (Block)
- 피해를 먼저 흡수. 초과분만 HP 감소
- **플레이어**: **라운드 종료 시 리셋** (한 라운드 내 4턴 동안 누적/유지)
- **보스**: 다음 보스 턴 시작 시 리셋 (미구현)

## 7. 라운드/턴 시스템

### 7.1 라운드 구조
- **4턴/라운드**
- **고정 교대**: 플레이어 → 보스 → 플레이어 → 보스 (셔플 없음)
- 예측 가능한 순서로 전략적 계획을 가능하게 함

### 7.2 턴 오더 UI
- 라운드 시작 시 4개 슬롯 모두 처음부터 공개 표시
- 현재 턴: 밝게 강조 / 지난 턴: 40% 투명 / 이후 턴: 70% 투명

### 7.3 플레이어 턴 흐름
1. AP 3으로 리셋
2. 장착된 모듈 훅 실행 (`on_player_turn_start`) — 예: 견고한 갑옷 → 방어도 +2
3. 페이즈 배너 표시
4. 파이프 앞 5장 → 손패로 드로우
5. 손패에서 드래그&드롭으로 카드 사용/버리기
6. 턴 종료 버튼 → 남은 손패 순서 지정 → 파이프 맨 뒤로

### 7.4 보스 턴 흐름
1. 파워 카드 카운트다운 틱 — 0이 된 파워 카드 즉시 발동 후 버린 카드 더미로
2. 덱 앞에서 카드 1장 드로우
3. 페이즈 배너 표시 (발동된 파워 + 드로우한 카드 내용)
4. 카드 실행
   - **ATTACK**: 효과 즉시 실행 → 버린 카드 더미
   - **POWER**: 파워 존에 배치 (카운트다운 토큰 부여) → 이후 매 보스 턴 -1 → 0 시 발동
5. 장착된 모듈 훅 실행 (`on_boss_turn_end`)
6. 자동으로 다음 턴 진행

## 7.5 보스 페이즈 시스템

### 개요
보스는 전투 중 **3단계 페이즈**를 거친다. 페이즈가 올라갈수록 마켓 카드 풀이 상위 티어로 이동한다.

### 전환 트리거
- **HP 임계 도달 시**만 발화 (라운드 트리거 없음)
- 단방향 — 페이즈는 올라가기만 함 (HP 회복 시 내려가지 않음)

| 페이즈 | HP 임계 |
|--------|---------|
| 2 진입 | ≤ 66% |
| 3 진입 | ≤ 33% |

### 평가 시점
- 보스 HP 변경 시 `_on_boss_hp_changed` → `phase_system.check_hp_trigger()` 호출

### UI
- `%PhaseLabel` 텍스트 "페이즈 N" + 색상(흰 → 청 → 빨)
- `phase_banner`로 "페이즈 N 진입!" 안내

### 구현
- `scripts/bosses/boss_phase_system.gd` — `BossPhaseSystem` (RefCounted, `GameContext` 주입)
- `GameContext`와 분리: 페이즈는 **보스 전용 상태**이므로 공유 컨텍스트에 추가하지 않음
- 시그널: `phase_changed(new_phase, old_phase)` → `MarketPanel.set_phase()`·UI 갱신

## 10.4 라운드 마켓

### 동작
- **라운드 시작 시**: 4개 레인 각각의 서브풀에서 **페이즈 가중치 추첨**으로 1장씩 진열 (총 4장)
- **구매**: 카드별 고정 골드 가격 소모 → 구매 즉시 **파이프 맨 뒤**에 추가. 해당 레인은 라운드 종료까지 비움
- **리롤**: 4레인 전체 동시 재추첨. 비용 — `AP 3` 또는 `골드 3` (횟수 제한 없음)
- **턴 제약**: 플레이어 턴에만 모든 마켓 버튼 활성화. 보스 턴/턴 종료 시 비활성

### 구현
- `scripts/ui/market_panel.gd` — `_load_pool(dir)`: `DirAccess`로 디렉토리 스캔 → 풀 자동 빌드 (씬 파일에 카드 등록 불필요)
- 리소스 디렉토리: `resources/cards/warrior/market/{attack,defense,special,gold}/`
  - 새 카드 추가 = 해당 디렉토리에 `.tres` 파일 추가만으로 자동 인식
  - 티어 구분은 `CardData.tier` 필드로만 처리 (디렉토리로 구분 안 함)
- `@export var {attack,defense,special,gold}_dir: String` — 씬에서 오버라이드 가능 (다른 직업 지원)
- 시그널: `card_purchased(card_data: CardData)` → `main_scene._on_market_card_purchased`가 파이프 맨 뒤로 인스턴스화

### 설계 의도
- **레인 보장**: 매 라운드 공격·방어·특수·골드 선택지가 항상 존재 → 덱 빌딩 방향성을 플레이어가 통제
- **골드 레인 루프**: 골드 카드 구매 → 파이프에 삽입 → N턴 후 손에 들어옴 → 그 라운드 마켓에서 T2 카드 구매 — 타임라인 설계와 결합된 경제 루프
- 리롤이 있어 운에 매몰되지 않으면서 AP 3 비용으로 전투 효율과 트레이드오프 발생

## 10.5 UI 피드백 시스템

### HP 바
- 플레이어·보스 HP 레이블 바로 아래에 9px 높이 컬러 바 표시
- HP 비율에 따라 색상 자동 변경: `> 60%` 초록 / `30~60%` 노랑 / `< 30%` 빨강
- `_create_hp_bar(label)`: 레이블 아래에 삽입, fill `ColorRect` 반환
  - 부모가 `PanelContainer`인 경우 조부모 레벨에 삽입 (PanelContainer 다중 자식 레이아웃 미지원)
- `_update_hp_bar(fill, current, max)`: anchor_right로 비율 표현

### 데미지 팝업
- `DamagePopup` (`scripts/ui/damage_popup.gd`, Label 상속)
- 피해: 빨간 `-N` / 회복: 초록 `+N` 숫자가 위로 떠오르다 0.9초 후 소멸
- 외곽선(outline_size=4) 처리로 배경 무관 가독성 확보
- `DamagePopup.spawn(parent, pos, amount, is_heal)` — 정적 팩토리 메서드
- 호출 시점: `_on_player_hp_changed` / `_on_boss_hp_changed`에서 이전 HP 비교
- 좌표: `to_local(label.get_global_rect().get_center())` — 화면 흔들림 중에도 정확한 로컬 좌표 보장

### 화면 흔들림
- 플레이어가 피해를 받을 때만 발동 (`intensity=7, duration=0.25s`)
- `_shake_screen()`: 메인 씬 루트 `position`을 Tween으로 랜덤 오프셋 후 복귀
- 진행 중 중복 흔들림 방지 (`_shake_tween` 추적 후 kill)
