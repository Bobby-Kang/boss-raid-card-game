## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 가제 | 프로젝트: 타임라인 |
| 장르 | 1:1 보스 레이드 카드게임 (턴제) |

| 핵심 메카닉 | 타임라인 파이프(FIFO 큐) / AP 자원 / 라운드 턴 오더 |

### 핵심 아이덴티티
> **"타임라인을 설계하라"**
> 카드는 파이프를 순환한다. 쓰거나 버린 카드는 파이프 맨 뒤로 돌아온다.
> 어떤 순서로 보내느냐가 미래를 결정한다.

## 3. 자원 시스템

### 3.1 AP (Action Point)
- **최대치**: 3
- **충전**: 플레이어 턴 시작 시 3으로 완전 리셋
- **소모**: 카드 사용 시 비용만큼 차감. AP 부족 시 사용 불가
- **이월 없음**: 남은 AP는 다음 턴으로 넘어가지 않음

### 3.2 골드
- **획득**: `GainGoldEffect` 카드 사용 시 증가
- **소멸**: 턴 종료 시 전량 증발
- **용도**: 라운드 마켓에서 카드 구매 (각 카드별 고정 가격) + 마켓 리롤(3 골드)

## 4. 타임라인 파이프 시스템

### 4.1 핵심 구조
```
[타임라인 파이프 / queue_cards[]]
  1번 카드 → 2번 카드 → 3번 카드 → ... → N번 카드
        ↓ (턴 시작, 앞 5장 드로우)
[손패 존 / hand_cards[]]
  카드A  카드B  카드C  카드D  카드E   ← 사용 가능
        ↓ (사용 or 버리기)
[타임라인 파이프 맨 뒤로 추가]
```

- `queue_cards[]`: 파이프 대기 중인 카드 (번호 순으로 표시)
- `hand_cards[]`: 현재 손패 (최대 5장, 드래그 조작 가능)
- 카드를 사용하거나 버리면 → 파이프 맨 뒤(Tail)로 추가
- 턴 시작 시 파이프 앞에서 5장 꺼내 손패로

### 4.2 FIFO 순환 원리
- 덱을 **섞지 않음**
- 버린 순서가 곧 다음에 드로우되는 순서
- 턴 종료 시 남은 손패는 **플레이어가 순서 지정** 후 파이프 맨 뒤로

### 4.3 UI 구성
| 영역 | 위치 | 내용 |
|------|------|------|
| 🃏 손패 | 하단 중앙 | 금색 테두리(`StyleBoxFlat_active`, 다크 판타지 톤), 드래그 가능한 손패 5장 |
| 📜 타임라인 파이프 | 하단 우측 | 미니 카드 행(비용 칩 + 카드명) 세로 스크롤. 행 호버 시 풀사이즈 카드 프리뷰 |

## 5. 카드 시스템

### 5.1 카드 타입

| 타입 | 설명 | 구현 상태 |
|------|------|----------|
| ATTACK | 피해 공격 카드 | ✅ |
| SKILL | 방어·유틸리티 카드 | ✅ |
| POWER | 지속 효과 카드 | 계획 중 |
| MODULE | 액티브 슬롯에 장착하는 영구 강화 카드 | ✅ (반격 태세 구현) |

### 5.2 카드 효과 클래스

| 클래스 | 효과 |
|--------|------|
| `DamageEffect(amount)` | 보스에게 `amount` 피해 |
| `BlockEffect(amount)` | 플레이어 방어력 +`amount` |
| `BlockDamageEffect` | 현재 플레이어 방어도만큼 보스에게 피해 (value 미사용) |
| `GainGoldEffect(amount)` | 골드 +`amount` |
| `DrawEffect(amount)` | 파이프 앞에서 `amount`장 즉시 손패로 |
| `ExileEffect(amount)` | 손패에서 `amount`장 선택해 **영구 소멸** (파이프 미복귀) |
| `RemoveCardEffect` | 손패 또는 파이프에서 1장 영구 제거 (선택 UI 통해) |

각 효과는 호버 프리뷰용 메타데이터를 `get_preview_summary() -> Dictionary` 가상 메서드로 노출. UI는 효과 클래스 이름·경로에 의존하지 않고 이 메서드 결과만 사용.

### 5.3 카드 키워드

| 키워드 | 효과 |
|--------|------|
| `consume = true` | 사용 후 파이프로 돌아가지 않고 **영구 소멸** (예: 마켓 *각인의 의식*) |

### 5.4 스타터 덱 (10장)

| 카드명 | 타입 | AP 비용 | 효과 | 수량 |
|--------|------|---------|------|------|
| 골드 카드 | SKILL | 0 | 골드 +2 | 5 |
| 베기 | ATTACK | 1 | 보스 피해 3 | 2 |
| 막기 | SKILL | 1 | 방어력 +3 | 2 |
| 집중 | SKILL | 0 | 카드 드로우 1 | 1 |

### 6.4 보스 페이즈

## 7. 라운드/턴 시스템

### 7.1 라운드 구조
- **4턴/라운드**
- **고정 교대**: 플레이어 → 보스 → 플레이어 → 보스 (셔플 없음)
- 예측 가능한 순서로 전략적 계획을 가능하게 함

### 7.2 턴 오더 UI
- 라운드 시작 시 4개 슬롯 모두 처음부터 공개 표시
- 현재 턴: 밝게 강조 / 지난 턴: 40% 투명 / 이후 턴: 70% 투명

### 7.3 플레이어 턴 흐름
1. AP 3으로 리셋
2. 장착된 모듈 훅 실행 (`on_player_turn_start`) — 예: 견고한 갑옷 → 방어도 +2
3. 페이즈 배너 표시
4. 파이프 앞 5장 → 손패로 드로우
5. 손패에서 드래그&드롭으로 카드 사용/버리기
6. 턴 종료 버튼 → 남은 손패 순서 지정 → 파이프 맨 뒤로

### 7.4 보스 턴 흐름
1. 파워 카드 카운트다운 틱 — 0이 된 파워 카드 즉시 발동 후 버린 카드 더미로
2. 덱 앞에서 카드 1장 드로우
3. 턴 인디케이터·보스 인텐트 갱신
4. **보스 행동 카드 연출** (`BossActionPresenter`) — 텍스트 배너 대신 실제 카드를 화면 중앙에 큰 카드로 제시
   - **ATTACK**: 보스 쪽에서 날아와 제시 → 플레이어 쪽으로 **슬램 + 임팩트(셰이크/플래시)** → 효과 적용(`resolve_cb`로 임팩트 순간 동기화) → 버린 카드 더미
   - **POWER**: 제시 후 **파워 존으로 흡수**되듯 축소 이동 → 파워 존 배치(카운트다운 토큰) → 이후 매 보스 턴 -1 → 0 시 발동
   - **발동된 파워(TRIGGER)**: "💥 파워 발동!" 헤더로 카드 슬램 연출
   - **무효화(NEGATED)**: *불멸의 방패* 등으로 막히면 "🛡 행동 무효화!" + 카드 흔들림 후 폐기
5. 장착된 모듈 훅 실행 (`on_boss_turn_end`)
6. 자동으로 다음 턴 진행

> 연출 흐름은 `await` 기반 — 카드 등장(0.35s) → 제시(0.7s) → 마무리 순으로 진행되며, 실제 피해/효과는 슬램 임팩트 순간에 적용되어 얼굴 피격 리액션과 동기화된다.

## 7.5 보스 페이즈 시스템

### 개요
보스는 전투 중 **3단계 페이즈**를 거친다. 페이즈가 올라갈수록 마켓 카드 풀이 상위 티어로 이동한다.

### 전환 트리거
- **HP 임계 도달 시**만 발화 (라운드 트리거 없음)
- 단방향 — 페이즈는 올라가기만 함 (HP 회복 시 내려가지 않음)

| 페이즈 | HP 임계 |
|--------|---------|
| 2 진입 | ≤ 66% |
| 3 진입 | ≤ 33% |

### 평가 시점
- 보스 HP 변경 시 `_on_boss_hp_changed` → `phase_system.check_hp_trigger()` 호출

### UI
- `%PhaseLabel` 텍스트 "페이즈 N" + 색상(흰 → 청 → 빨)
- **페이즈 전환 시네마틱** (`_play_phase_transition_fx`): 페이즈 색 화면 섬광 + 강한 화면 흔들림 + 보스 얼굴 각성(스케일 펀치 + 밝은 틴트 펄스) + 큰 컬러 타이틀(`phase_banner.show_title`, `⚔ PHASE N ⚔` + 부제 + 스케일 팝)

### 구현
- `scripts/bosses/boss_phase_system.gd` — `BossPhaseSystem` (RefCounted, `GameContext` 주입)
- `GameContext`와 분리: 페이즈는 **보스 전용 상태**이므로 공유 컨텍스트에 추가하지 않음
- 시그널: `phase_changed(new_phase, old_phase)` → `MarketPanel.set_phase()`·UI 갱신

## 7.6 버그베어 카드 풀 (18장)

> *"피 냄새 맡은 야수 — HP가 깎일수록 더 위험해진다."*

### Phase 1 (6장) — 탐색 + 위협 학습

| 카드 | 타입 | 카운트 | 효과 |
|---|---|---|---|
| 할퀴기 | ATTACK | — | 플레이어 4 피해 |
| 으르렁 | ATTACK | — | 3 피해 + **드로우 봉인 1** |
| **웅크리기** ⭐ | POWER | 3 | 💎즉시 방어 +5 / ⏳매턴 방어 +3 / 💥 방어도만큼 반사 피해 |
| 맹독 발톱 | ATTACK | — | 4 피해 + **취약 1** |
| **위협** 🆕 | POWER | 2 | 💥 3 피해 + 보스 HP +3 회복 (흡혈) |
| **사냥의 시작** 🆕 | POWER | 3 | 💥 **피 냄새 활성** (HP 50% 이하 시 모든 공격 ×1.3) |

### Phase 2 (6장) — 본격 위협

| 카드 | 타입 | 카운트 | 효과 |
|---|---|---|---|
| 강타 | ATTACK | — | 8 피해 |
| 분노의 포효 | POWER | 2 | 💥 **보스 공격력 +2 영구** |
| **강철 벽** ⭐ | POWER | 2 | 💎즉시 방어 +6 / ⏳매턴 방어 +4 / 💥 방어 +8 |
| 연속 할퀴기 | ATTACK | — | 3 피해 × 3회 (다단) |
| **잔혹한 일격** 🆕 | ATTACK | — | 6 피해 + **취약 2** |
| **야수의 외침** 🆕 | POWER | 1 | 💥 **모든 활성 POWER 카운트 -1** |

### Phase 3 (6장) — 광폭 결정타

| 카드 | 타입 | 카운트 | 효과 |
|---|---|---|---|
| 광란 | ATTACK | — | 10 피해 + 보스 자해 -3 |
| 분쇄 | ATTACK | — | 8 피해 + **드로우 봉인 2** |
| 절망의 포효 | POWER | 1 | 💥 8 피해 + 드로우 봉인 2 |
| 최후의 발악 | POWER | 3 | 💥 **16 피해** (결정타) |
| **광기의 돌진** 🆕 | ATTACK | — | 11 피해 |
| **피의 추격** 🆕 | ATTACK | — | 플레이어 HP ≤30% → **15 피해**, 아니면 5 피해 |

⭐ = POWER 다단 효과 / 🆕 = 신규 / 💎 즉시 / ⏳ 매턴 / 💥 카운트 0

### 신규 디버프·상태 시스템

#### 플레이어가 *받는* 디버프
| 상태 | 효과 | 소진 |
|---|---|---|
| **드로우 봉인 N** | 다음 플레이어 턴 드로우 -N | 1회성 (소비 후 0 리셋) |
| **취약 N스택** | 받는 다음 N번 피해 ×1.5 | 스택당 1회 |

#### 보스 자체 상태
| 상태 | 효과 | 트리거 |
|---|---|---|
| **피 냄새** | HP ≤50% 시 모든 ATTACK ×1.3 | *사냥의 시작* 발동 후 영구 |
| **공격력 보너스** | 모든 ATTACK +N 영구 | *분노의 포효* 발동 시 |

### POWER 다단 효과 시스템

> *"POWER 카드는 *드로우 시* + *매턴* + *카운트 0* 3단계로 작동한다."*

`BossCardData` 확장:
```gdscript
@export var on_draw_effects: Array[CardEffect] = []   # POWER 드로우 시 즉시
@export var on_tick_effects: Array[CardEffect] = []   # POWER 매 보스 턴
@export var effects: Array[CardEffect] = []           # 카운트 0 시 발동 (또는 ATTACK 즉시)
```

`BossDeckSystem`:
- `play_card`에서 POWER 카드일 때 `on_draw_effects` 즉시 실행
- `tick_powers`에서 매 보스 턴 시작 시 `on_tick_effects` 실행 → 카운트 -1
- 카운트 0 도달 시 `effects` 실행 → 버린 카드 더미

**결과**: 깎아서 끊어도 *드로우 시 효과는 이미 발동*, 안 끊으면 *카운트 0 큰 효과* — 진짜 선택지

### 페이즈 가시화 (보스 덱 시각 분리)

덱 자체는 `[P1 셔플][P2 셔플][P3 셔플]` 순서로 쌓이지만(티어 경계 고정), 카드 인스턴스 자체에는 페이즈 정보가 없으므로 **`BossDeckSystem._card_phase` 사이드-딕셔너리**로 출처 페이즈를 추적한다.

- `get_phase_of(card) → int` — 카드의 출처 페이즈 (1/2/3)
- `get_remaining_counts_by_phase() → {1, 2, 3: n}` — 페이즈별 남은 장수
- `get_remaining_names_by_phase() → {1, 2, 3: [names]}` — 칩 툴팁용

**UI 표현:**
- **3-페이즈 칩 행** (`BossInfoRight` 상단): `P1·n / P2·n / P3·n` — 각 칩이 페이즈 색(P1 양피지·P2 청·P3 적)으로 표시되고, 소진된 페이즈는 회색 디밍. 호버 시 해당 페이즈 남은 카드 목록 툴팁. 기존 `🂠 18` 단일 라벨을 대체.

## 10.4 라운드 마켓

### 동작
- **모달 토글**: 상단 슬림바 `🛒 상점` 버튼으로 열고 닫는 중앙 팝업 오버레이. 평소 숨김 → 화면 공간을 1:1 대결 무대에 할애. 배경 딤 클릭 또는 `✕ 닫기`로 닫힘
- **라운드 시작 시**: 4개 레인 각각의 서브풀에서 **페이즈 가중치 추첨**으로 1장씩 진열 (총 4장)
- **구매**: 카드별 고정 골드 가격 소모 → 구매 즉시 **파이프 맨 뒤**에 추가. 해당 레인은 라운드 종료까지 비움
- **리롤**: 4레인 전체 동시 재추첨. 비용 — `AP 3` 또는 `골드 3` (횟수 제한 없음)
- **턴 제약**: 플레이어 턴에만 모든 마켓 버튼 활성화. 보스 턴/턴 종료 시 비활성
- **페이즈 연동**: `phase_changed` 시그널 수신 시 `set_phase(new_phase)` → 다음 `refresh_slots()`부터 가중치 적용

- 좌표: `to_local(label.get_global_rect().get_center())` — 화면 흔들림 중에도 정확한 로컬 좌표 보장

### 화면 흔들림
- 플레이어가 피해를 받을 때만 발동 (`intensity=7, duration=0.25s`)
- `combat_fx.shake_screen()`: 메인 씬 루트 `position`을 Tween으로 랜덤 오프셋 후 복귀
- 진행 중 중복 흔들림 방지 (Tween 추적 후 kill)

### 캐릭터 피격 리액션
- 보스/플레이어 얼굴이 피해를 받으면: 빨간 플래시(`Color(2.2, 0.45, 0.45)`) + 스케일 펀치 + ±0.18° 기울임 + 복귀
- 위치 대신 `scale`/`rotation` 사용 → Container 자식 노드에서도 안정적
- `combat_fx.flash_recoil(target, direction_x)`

### Hit-stop (명중 정지)
- 데미지 발생 순간 `Engine.time_scale = 0.05`로 0.04초간 시간 정지 → 1.0 복귀
- 타격감 폭발적 향상, 셰이크/플래시/팝업과 동기화
- `combat_fx.hit_stop()` (재진입 방지 플래그 + `ignore_time_scale=true` 타이머)

### UX 보조 시스템

**드롭존 가시화**
- 카드 드래그 시작 시 모든 드롭존에 라벨 + 색 오버레이 (`drag_label_text` export)
- 수용 가능: 청색 펄스 (`accept_filter` 통과)
- 수용 불가: 빨간 디밍 + `✕` 마커
- `pass_through_when_idle = true`로 평소엔 클릭 통과

**카드 호버 프리뷰**
- 손패/파이프 카드에 호버 시 효과 미리보기 부동 라벨 스폰
  - 보스 HP 옆: `−N` (보스 방어도 차감 반영)
  - 플레이어 블록 옆: `+N 🛡`
  - HP 옆: `+N 드로우` / `+N 💰` / `✖ 카드 정리`
- 효과 식별은 `CardEffect.get_preview_summary()` 가상 메서드 결과 사용 (효과 클래스 추가/리네임에 강함)
- 파이프 행 호버 시 추가로 **풀사이즈 카드 한 장**이 행 우측에 페이드인 (`CardScene` 인스턴스, mouse_filter=IGNORE)

**턴 인디케이터**

- 내 턴: `🔵 내 턴` (청색) + 힌트 `카드를 드래그하거나 턴 종료`

**컨텍스트 디밍**

- 내 턴 복귀 시 1.0

## 10.6 보상형 카드 영구 제거

자동 소멸 시스템 폐지 → 빌드 게임 본질에 맞는 *보상형 도구*로 전환.

### 두 트리거
1. **페이즈 전환 보상** — 보스 HP가 66% / 33% 임계 돌파 시 1회씩 자동 발생
   - 배너: `🌟 전사의 깨달음 — 보스의 약점이 보인다` 안내 후 선택 오버레이
   - 페이즈 전환 시그널 콜백 도중 await race를 피하기 위해 **`_pending_rewards` 큐**에 enqueue → `_begin_player_turn` 시작 시 안전하게 처리
2. **마켓 *각인의 의식*** (`market_engraving.tres`, 5골드, T2 특수 레인)
   - 1 AP 사용 → 손패/파이프 카드 1장 영구 제거
   - `consume = true` — 사용 후 카드 자체도 영구 소멸 (인플레 방지)

### 공통 흐름
- `_grant_card_removal_reward(reason_lines)` — 손패+파이프 통합 선택 UI
- 1장 선택 또는 **스킵 가능** (강제 X)
- 선택 시 `exile_animator.play(card)` — 슬라이드 → 흔들림 → 찢어짐 + "✖ 영구 소멸" + "💀 {카드명}이 영원히 사라졌다"

### 디자인 의도
- 빌드 게임에서 *카드를 잃는 것*은 *덱이 더 좋아지는 길*일 때 빛난다
- 페이즈 전환 = 게임 진행도 = 카드 제거 보상 — 서사 정합성
- 자원 통합: 골드 → 마켓 → 각인의 의식 → 빌드 정제 (새 자원 루프)
