## 2. 고유 능력: 투기 발산 (Innate Ability) ✅ 구현 완료

카드를 사용하지 못해 버려지는 AP를 자원으로 치환하여, 패가 꼬인 상황을 타개하는 안전망이자 궁극기 시스템.

### 패시브 — 투기 축적
- 플레이어 턴 종료 시, 사용하지 않고 남은 AP 1당 **투기 스택 1** 획득
- 최대 10스택

### 액티브 — 투기 발산 (0 AP)
- 투기 스택이 **10**에 도달하면 활성화
- **효과:** 보스에게 즉시 **10 피해** + 플레이어 **방어도 10** 획득
- 발동 후 스택 0으로 초기화

### 구현
- `scripts/characters/warrior/rage_system.gd` — `WarriorRageSystem` (RefCounted, `GameContext` 주입)
- UI: `BuffBar` 내부 `RageLabel` + 10칸 `RageOrbs` + `RageButton` (ResourceBar 우측)
- 트리거: `main_scene._finish_turn()`에서 `rage_system.add(ap_manager.current)` 호출

**설계 의도:** 카드 직접 사용(1 AP → 2~3 가치)보다 낮은 효율(1 AP → 평균 2 가치 분산)로 설정.
카드를 쓰는 것이 항상 최선이되, 버려지는 AP가 없도록 보완한다.

## 4. 고유 시작 모듈 ✅ 구현 완료

전투 시작 시 액티브 슬롯 1에 기본 장착된 패시브 아이템.

### [모듈] 반격 태세
- **AP:** 0 (전투 시작 시 장착 상태)
- **효과:** 보스의 턴이 끝날 때, 플레이어 방어도가 1 이상 남아있으면 보스에게 **2 피해**

### 구현
- `scripts/characters/warrior/counter_stance_ability.gd` — `ModuleAbility` 상속, `on_boss_turn_end` 오버라이드
- 리소스: `resources/cards/warrior/module_counter_stance.tres`
- 장착: `main_scene._equip_warrior_modules()`가 시작 시 `ActiveSlot1`에 배치 (`set_active(false)`로 드래그 잠금)
- 트리거: `_begin_boss_turn()`에서 보스 피해 처리 후 `active_cards` 순회하며 `on_boss_turn_end(ctx)` 호출

**설계 의도:** 정밀한 방어 계산에 대한 보상. 소량 피해라도 누적시켜 전투를 유리하게 이끈다.
방어도가 **라운드 종료 시점까지 유지**되도록 변경되면서 (2026-04-15) 반격 태세의 발동 기회가 크게 늘어났다.

## 5. 핵심 시스템 정의

### 골드 소멸
턴 종료 시 남은 골드는 전량 증발한다.
이번 턴에 골드 카드를 몇 장 뽑아 마켓의 어떤 카드를 살지 **타이밍**을 설계해야 한다.

### 방어도 — 라운드 유지
플레이어 방어도는 **라운드 종료 시점**까지 유지된다 (5턴 내내 누적).
이로써 막기 카드는 축적 가능한 자원이 되고, 반격 태세 모듈과 투기 발산의 방어도 보너스가 모두 온 라운드에 걸쳐 의미를 가진다.

### 타임라인 프로그래밍
턴 종료 시 사용/남은 카드를 원하는 순서대로 정렬하여 파이프 맨 뒤에 추가한다.
버린 순서 그대로 다시 드로우되므로 **지금 버리는 순서가 미래를 결정한다.**
